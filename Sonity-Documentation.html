System Architecture

The Sonity platform follows a modern microservices-inspired architecture with clear separation of concerns:

  • Frontend Layer: Next.js application with React components
  • API Layer: RESTful API routes within Next.js
  • Data Layer: MongoDB with Mongoose for data modeling
  • Audio Layer: HLS.js for streaming and audio processing
  • State Layer: React Context and custom hooks

API Documentation

API Endpoints

Subscription API

GET /api/v1/subscription - Returns default free subscription - No authentication required - Response: { plan: "free", status: "active" } POST /api/v1/subscription/upgrade - Creates upgrade order (disabled) - Returns "Coming Soon" message - Razorpay integration ready POST /api/v1/subscription/cancel-queued-change - Cancels scheduled changes - Returns success for all users

Capabilities API

GET /api/v1/capabilities - Returns user capabilities - Default: Basic features enabled - No authentication required

Response Format

interface APIResponse<T> { success: boolean; data?: T; error?: string; message?: string; }
